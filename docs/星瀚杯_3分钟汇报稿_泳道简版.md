## 图1 · 系统流程与功能总览（约 90 秒）

请老师看**第一张图「系统流程与功能总览」**——这是**泳道图**，从上到下四层：**业务层、法务层、AI 层、输出层**。

**业务层**，① **冷启动**：新用户做 Playbook 访谈，定行业、风险偏好和模板风格，不用先懂巴西法条。  
② **提交材料**：合同、尽调清单等进**同一项目**。  
③ **定范围**：法务勾选协查项，可调**匹配度阈值**和**每条检索条数 Top-N**。

**法务层**，④ **协查包**：按 Playbook 生成结构化问题与检索计划。  
⑤ **Gate A**：系统先给 Top 法条与摘要，**法务确认或驳回**后才进入生成——这是**人机协同**，不是黑盒直出。  
⑥ **复核定稿**：改结论、补批注，系统记录**偏好**。

**AI 层**（与主流程并行）：**合同审查、尽调核查、项目统摄**三条线共用同一语料与 Playbook。

**输出层**，⑦ **定稿导出**：协查 Word、合同表、尽调表、冲突表——格式固定，**可对外引用、可签字**。

右侧红色虚线是**法规监测**：语料 Agent 跟踪 LexML 等更新；**已定稿文档不因新法自动改写**，只提醒订阅。

图例里还有 **Playbook 偏好闭环**：您在复核里的确认、阈值和 Top-N 选择，会写入**个人偏好**，下次同类型协查**自动沿用**——这是后面图2会展开的核心能力。

## 图2 · 差异化能力（约 80 秒）

**第二张图「差异化能力」**，九项，三行三列。

**01 项目制工作台** — 协查、合同、尽调、冲突一张表，不用切多个工具。  
**02 Playbook 冷启动** — 访谈 + 可选上传制度文件，生成组织级 Playbook，降低巴西法冷启动成本。  
**03 可引用 · 可签** — 交付物带法条引证链与免责声明，满足对外协查与内控留痕。

**04 Playbook 偏好闭环** — 这是**根据法务个人行为持续修正系统行为**的机制：  
您在 Gate A **确认或驳回**某条法条、调整**匹配度阈值**、改 **Top-N**、或在 Playbook 上产生**偏差记录**，系统都会记入**个人偏好**；下一次同类协查**默认按您的习惯检索与排序**，组织 Playbook 也可随偏差迭代。一句话：**越用越像本团队法务，而不是通用 ChatGPT**。

**05 统摄 + Red Team** — 项目级 Analyst 汇总 + Red Team 挑漏洞，跨模块看风险，不是单点问答。  
**06 法源常新** — LexML · Planalto · STF · gov.br 官方法源链 + 语料 Agent，引证可回溯。

**07 增量协查** — 材料或法规变化时，只重跑受影响条目，不全量重做。  
**08 多源法条引证链** — 语料 → LexML URN → Planalto / STF / gov.br 门户，三层可点击核验。  
**09 跨模块冲突检测** — 合同结论、尽调项、协查结论不一致时自动标红，避免「各说各话」。
