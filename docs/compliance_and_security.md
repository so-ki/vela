# Vela 出海法务台 · 合规与安全说明

> 面向星瀚杯 / 产品评审提交的合规与安全能力说明（MVP 实现边界见各节标注）。

## 1. 产品定位与免责

- Vela 是**法务协查辅助工具**，输出核查清单、多源 RAG 引用与双语简报草稿，**不构成法律意见**。
- 所有对外交付物须经**法务人工复核**（清单条目确认 / 驳回）后定稿；系统默认阻断「匹配度低于阈值」的条目自动通过。
- 导出 Word/PDF 含固定免责声明，提示用户 consult qualified local counsel。

## 2. 人机协同与门控

| 门控 | 说明 |
|------|------|
| Gate A | 材料构成要件 + RAG 匹配度聚合；未通过则锁定清单复核 |
| 匹配度阈值 | 法务在确认协查范围时可配置（50–95，默认 70） |
| 清单复核 | 逐条确认 / 驳回；全部处理完方可定稿 |
| 定稿版本 | 格式 `v{修订轮次}.{定稿序号}`，如 v1.0、v1.1 |

## 3. 数据与权限

- **角色分离**：业务（材料提交）、法务（范围确认、Gate A、复核、导出）、管理员（语料维护）。
- **审计日志**：关键操作（生成协查包、打回材料、复核定稿、语料变更）写入 audit log。
- **材料存储**：上传文件归档于服务端受控目录；API 需认证，场景按创建人 / 法务可见性隔离。
- **敏感信息**：MVP 建议部署于内网或私有云；生产须启用 HTTPS、密钥管理与数据库加密（文档级要求，部署时实施）。

## 4. 法源与 RAG 安全

- 语料来源标注 LexML / STF / STJ / Jusbrasil 等渠道；检索结果携带 **URN、有效性（vigente / revogado）**。
- 已废止（`revogado` / `repealed`）条目在检索层过滤，避免引用失效法条。
- 二次扩展检索（功能三↔二回路）在低于阈值时触发，仍受同一阈值门控。
- 语料维护与索引重建需法务角色；监测辅线检测本地语料 Diff，**不自动改写已定稿**。

## 5. 红旗与一致性（非阻断）

- 对材料中「无环保处罚」等表述与公开执法线索进行**提示级**比对（MVP 为演示占位数据）。
- 用工人数与描述文本不一致等轻量一致性检查，结果展示于 Gate A，供法务决策。

## 6. 增量协查与版本

- 业务补材料后支持**增量重算**关联核查项，未变更条目沿用上一轮 RAG / 简报 / 复核结论。
- 法规语料变更通过订阅 + Diff 提醒受影响核查项编号，由法务决定是否重跑协查。

## 7. MVP 边界（未声称已生产就绪）

- 法源为 **~70 条精选语料** + LexML URN **按需 HTTP 拉取**（非全库实时镜像）；监测为本地语料指纹 + 手动扫描。
- 匹配度为 **关键词 + 维度加权**（可选 Chroma），详见 [`match_tier_and_gate.md`](./match_tier_and_gate.md)。
- 未实现完整个人数据跨境传输评估流程；客户部署时需按属地法补充 DPIA / 数据出境评估。
- 未集成 HSM / 国密；企业版可按客户安全基线扩展。

## 8. 联系与迭代

- 产品团队负责合规说明与伦理声明的同步更新。
- 重大模型或语料策略变更须更新本文件并通知法务负责人。
