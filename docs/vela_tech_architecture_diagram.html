复赛三条技术路径

路径 A · 法源常新（数据可信）
  • LexML 官方文本批量回溯 + 缓存，降低「法条过时」
  • Reg Feed 从「能打开网页」升级为「抓到变更摘要」
  • 语料变更 → 自动通知受影响项目 / 合同 / 尽调
路径 B · 智能加深（Agent 可交付）
  • LLM Analyst + Red Team 多轮辩论（非纯规则）
  • 冷启动上传 docx → 自动抽取 house rules / 底稿样例
  • 合同 / 尽调结果变更 → 触发协查增量重跑
路径 C · 工程可试点（稳定上线）
  • PostgreSQL + 云部署 + LexML 调用限流缓存
  • E2E 全绿 · 飞书 / SSO 可选接入
  • 星瀚 Playbook 模板驱动 Word 导出版式

并行辅线 · 法规动态监测

CAMEX · IBAMA · ANPD · 州法激励等官方源站轮询 + 本地语料 Diff；不自动改已定稿，只推法务复核。
