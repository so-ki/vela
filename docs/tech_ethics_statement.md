# Vela 出海法务台 · 技术伦理声明

## 1. 设计原则

我们遵循「**法务主导、AI 辅助、可审计、可拒绝**」原则开发 Vela：

1. **人类最终决策**：AI 不替代律师判断；任何协查结论须经法务清单复核后定稿。
2. **透明可解释**：每条核查项展示 RAG 命中来源、匹配度分数及是否需复核。
3. **最小必要自动化**：业务侧仅提交材料；范围、阈值、打回、导出由法务控制。
4. **失败安全**：Gate A 未通过或匹配度低于阈值时，系统默认**阻断**进入清单通过态。

## 2. 公平与偏见

- 规则包与语料按**巴西属地法**与行业场景组织，避免单一来源偏见；多源检索（联邦法、判例占位、州法条目）并列呈现。
- 不对用户种族、国籍等 protected attributes 做自动化评分；协查维度由法务勾选，不由模型擅自扩大范围。
- 已知风险：葡语语料覆盖不均、判例摘要质量参差——通过法务复核与语料维护缓解。

## 3. 隐私与尊严

- 项目材料可能含商业机密；系统按角色限制访问，并记录操作审计。
- 不向第三方模型提供商发送客户标识符（部署配置应使用脱敏或私有化推理，MVP 演示环境除外）。
- 用户有权通过业务流程撤回或补充材料；打回与修订轮次可追溯。

## 4. 可靠性与幻觉控制

- RAG 输出必须绑定语料片段与 URN；简报生成受清单条目与检索结果约束。
- 低于阈值的命中强制标注「需法务复核」，禁止 UI 暗示「已通过法律审查」。
- 红旗提示（材料 vs 公开线索）仅为**辅助线索**，算法不自动作出「虚假陈述」法律认定。

## 5. 法规监测伦理

- 监测辅线用于提醒语料/法规变更，**不自动修改已定稿协查包**。
- 场景订阅仅关联核查项编号，避免过度推送；法务决定是否重跑 RAG 或通知业务。

## 6. 责任边界

| 主体 | 责任 |
|------|------|
| 产品 / 技术团队 | 系统安全、门控逻辑、审计、伦理声明维护 |
| 客户法务 | 范围确认、阈值设定、复核定稿、对外法律意见 |
| 业务用户 | 材料真实、完整、及时更新 |

## 7. 持续改进

- 新功能上线前评估：是否削弱法务门控、是否引入不可解释自动化、是否扩大数据收集范围。
- 收到偏见、幻觉或隐私投诉时，优先关闭相关自动化路径并复盘语料与提示词。
- 与本项目《合规与安全说明》（`compliance_and_security.md`）配套使用，随版本迭代更新。

---

*本声明适用于 Vela 出海法务台 MVP 及星瀚杯方案演示，正式商用须结合客户法务与数据保护要求另行签署。*
