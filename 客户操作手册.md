# Vela 出海法务平台 · 客户使用手册

**适用对象：** 企业法务部、合规团队、投资协查业务人员  
**当前法域：** 巴西（演示行业：新能源投资设厂）  
**文档性质：** 面向最终用户，说明「怎么做、会得到什么、需要注意什么」

> 系统安装、账号开通、API 对接等技术事项由贵司 IT 或 Vela 实施人员负责。本文仅说明**登录后的业务操作**。

---

## 一、平台能帮您做什么

Vela 将您用**中文**描述的投资场景，转化为一套可复核、可溯源、可归档的协查材料：

| 交付物 | 说明 |
|--------|------|
| 《专项核查清单》 | 按劳工、外资、税制、行业准入等维度拆分的核查条目 |
| 法条检索结果 | 每条清单关联 LexML / 最高法院 / 高级司法法院等开放法源片段 |
| 《双语法律风险简报》 | 中文 + 葡萄牙语执行摘要与分维度风险说明 |
| 《法律研究意见书》（Word） | 经法务复核定稿后导出，默认法学院格式，便于二次编辑 |
| 《协查底稿》（PDF） | 经法务复核定稿后导出，便于传阅定版 |

**平台不能替代律师。** 所有输出均为协查线索与信息摘要，重大决策须由持证律师复核或出具正式法律意见。

## 二、使用前请知悉

### 2.1 三条核心原则

1. **有人工复核环节** — 系统不会自动「定稿」；须法务逐条确认或驳回后才能导出正式底稿。  
2. **有匹配度门控** — 法条匹配度低于 **70 分** 的条目，会强制标注「需法务复核」，不得当作已核实结论使用。  
3. **AI 仅辅助表述** — 若启用智能润色，仅优化中葡表述，**不新增**法律结论、法条名称或溯源链接。

### 2.2 数据与隐私

- 请用**业务语言**描述项目，避免上传不必要的个人身份信息或核心商业机密。  
- 可对敏感字段做脱敏后再输入（如具体金额、内部代号等）。  
- 首次使用须阅读并确认**免责声明与数据使用条款**。

### 2.3 您需要准备什么

| 准备项 | 建议 |
|--------|------|
| 登录地址 | 由管理员提供（通常为内网或试用环境 URL） |
| 账号密码 | 由管理员按角色开通（业务 / 法务） |
| 项目信息 | 项目名称、投资地、行业、投资结构、产能/用工等 |
| 业务描述 | 3～5 句中文，说明「做什么、在哪里、规模多大」 |
| 可选 | 若需智能润色简报，由管理员预先配置 API（用户无需自行操作） |

### 2.4 账号角色（业务 / 法务）

平台支持**分角色协作**：

| 角色 | 典型用户 | 主要权限 |
|------|----------|----------|
| **业务协同** | 投资部、项目部 | 提交场景、生成清单与简报、**提交法务复核** |
| **法务复核** | 企业法务部 | 查看全部待办、逐条复核、定稿、**导出 Word/PDF**、**维护法源语料** |

演示环境账号（密码均为 `Demo1234!`）：

| 角色 | 邮箱 |
|------|------|
| 业务 | `biz@demo.vela` |
| 法务 | `legal@demo.vela` |

## 三、登录与进入系统

### 您需要做什么

1. 在浏览器中打开管理员提供的地址。  
2. 输入邮箱和密码，点击登录。  
3. 若弹出**免责声明**，请完整阅读后点击确认。  

### 完成后您会看到什么

- 进入**工作台**首页。  
- **业务 / 法务**顶部导航：**工作台**；法务账号另可见 **法源维护**。业务提交新项目、法务待办复核均在**工作台页面内**操作。
- 页面显示欢迎语、法域（巴西）及角色对应的工作台入口。  
- **法务**另可见系统状态（规则库、法源索引）与法源维护入口。

### 请注意

- 未确认免责声明前，无法正常使用协查功能。  
- 演示环境：业务 `biz@demo.vela`、法务 `legal@demo.vela`，密码均为 `Demo1234!`（以管理员通知为准）。  
- 建议使用 Chrome、Edge 或 Safari 最新版；勿在公共电脑上勾选「记住密码」。

| 步骤 | 操作 |
|------|------|

| 2 | 点击 **「一键生成完整样本」** |

### 方式 B：逐步完成真实协查（推荐正式使用）

以下按业务顺序说明每一步。

---

## 步骤 1 · 进入工作台，了解当前任务

### 您需要做什么

登录后停留在**工作台**，查看：

- **业务账号：** 顶部欢迎语 + **「提交新协查」** 入口 + **「我的协查进度」**  
- **法务账号：** **「待办队列」**、**「合规审查维度」**、**「系统状态」**、**「法源维护」**、法规动态监测  

### 完成后您会看到什么

- 可从「最近协查场景」继续未完成的项目（进度 / 补充材料 / 查看法务反馈）。  
- **法务**可在「系统状态」确认规则库、法源索引是否就绪；**业务**无需关注后台索引细节，提交失败时联系管理员即可。

### 请注意

- **「演示项目一键提交法务」** 使用 BYD 坎皮纳斯预设场景，不代表贵司真实项目。  
- 若提交失败，通常是服务未启动，请联系管理员，**不要重复频繁点击**。

---

## 步骤 2 · 提交协查场景

**入口：** 工作台 **「填写项目并提交法务」**

### 您需要做什么

填写协查表单，核心字段如下：

| 字段 | 填写说明 |
|------|----------|
| 项目名称 | 必填，如「XX 公司圣保罗州工厂项目」 |
| 投资结构 | 如「100% 外资全资子公司」「中外合资 51%」 |
| 国家 / 州 / 城市 | 当前版本以巴西为主；演示模板为圣保罗州坎皮纳斯 |
| 行业 | 选择与您项目最接近的行业（演示为新能源） |
| 动作类型 | 如绿地建厂、收购现有资产等 |
| 雇员规模 | 预计本地用工人数 |
| 业务描述（中文） | **至少 10 个字**，建议 3～5 句说明投资背景与合规关注点 |
| 产能 / 厂房说明 | 可选，有助于生成更贴合的清单 |
| 协查范围 | **无需勾选** — 业务提交时默认覆盖全部 4 个合规维度（劳工、外资、税制、行业准入） |
| 时间线 | 董事会日期、开工、投产等，可选 |

填写完成后，点击 **「生成专项核查清单」**（业务账号为 **「生成并提交法务」**）。

**可选：上传投资方案**

- 在提交页上传 **.txt / .md / .docx**（≤2MB）  
- 系统自动抽取项目名称、用工、产能等**客观事实**预填表单  
- **不会**自动生成法律结论；提交前请人工核对数字与地点  
- 若管理员已配置 LLM，优先使用 AI 抽取；否则使用规则抽取

**快捷方式（提交页内）：**

- **「填入 BYD 演示模板」** — 仅用于学习表单结构，正式项目请改为您自己的内容。  
- **「演示项目一键提交法务」** — 工作台或提交页均可使用；跳过表单，直接用演示场景提交法务（同演示用途）。

### 完成后您会看到什么

- 系统自动跳转到 **《专项核查清单》** 页面。  
- 清单标题含项目名称，显示识别出的行业、动作类型。  
- BYD 一键样本约 **24 条**核查项（含电动客车 / 电池 / 光伏 / 储能 / 研发子赛道），分布在 4 个维度下。  
- 每条包含：编号、优先级、核查要点、**纳入理由**（说明为何需要查这一项）。  
- 若启用了智能意图解析，系统会结合您的中文描述补充审查侧重点（后台完成，页面不单独展示步骤）。

### 请注意

- **业务描述越具体，清单越贴合**；过于笼统的描述可能导致条目偏通用。  
- 国家/州/城市在部分环境下为只读（演示包固定巴西），正式多法域版本上线前请以实际表单为准。  
- 提交后场景会保存在「最近协查场景」中，可随时返回继续。

## 步骤 3 · 查看核查清单与法条依据

**入口：** 提交场景后自动进入，或工作台 → 某项目 → **清单**

### 您需要做什么

1. **等待法源检索完成**（页面加载时会自动进行，通常数秒至十几秒）。  
2. 逐条阅读 15 项核查内容。  
3. 展开每条下的 **「相关法条（Top 3）」**，查看匹配度与摘录。  
4. 需要核实原文时，点击 **「溯源：LexML / 法院门户 ↗」** 在新窗口打开官方法源。  
5. 阅读完毕后，点击 **「生成双语简报」**。

### 完成后您会看到什么

| 页面元素 | 含义 |
|----------|------|
| 匹配度分数 | 该法条与核查条目的相关程度（满分 100） |
| 「需法务复核」标签 | 匹配度 **低于 70 分**，不可视为已核实 |
| 法条摘录（葡语 / 中文） | 来自开放法源的片段摘要 |
| 来源标签 | LexML、STF、STJ、Jusbrasil 等 |
| 未命中提示 | 某条清单未检索到法条时，会提示建议扩大检索或外聘律所 |

页面顶部会显示类似：**「已绑定 X 条法源片段」**。

### 请注意

- **匹配度高 ≠ 法律结论成立**，仍须结合项目事实与律师意见判断。  
- 未命中法条的条目**不代表无风险**，往往意味着需人工补充检索。  
- 法条链接指向第三方官方网站，打开速度取决于网络环境。  
- 须等检索完成后再点「生成双语简报」；检索进行中按钮可能不可用。

## 步骤 4 · 生成并阅读双语法律风险简报

**入口：** 清单页 → **「生成双语简报」**，或工作台 → **简报**

### 您需要做什么

1. 进入简报页后，系统通常会自动生成第一版（若已有一键样本则直接展示）。  
2. 阅读 **执行摘要**（中文 + 葡萄牙语）。  
3. 按维度查看 **综合风险等级**（较高 / 中等 / 可控）及每条 **风险说明**。  
4. 核对 **法条依据** 与溯源链接。  
5. 查看 **「需法务复核清单」** 区域（若有未过门控的条目）。  
6. 根据需要选择生成方式：  
   - **「LLM 润色生成」** — 中葡表述更流畅（需管理员已配置 API，约 1～2 分钟）  
   - **「仅模板」** — 即时生成，不调用外部 AI  
7. 确认无误后：
   - **业务账号**：点击 **「提交法务复核」**（不再自行定稿导出）  
   - **法务账号**：点击 **「进入法务复核」**

### 完成后您会看到什么

| 内容 | 说明 |
|------|------|
| 简报状态 | 「可提交法务复核」/「部分条目需复核」/「暂不可自动定稿」 |
| 门控统计 | 已通过 X 条、需复核 X 条；阈值 70 分 |
| 生成模式 | 「LLM 润色」或「规则汇编」 |
| 分维度风险说明 | 每条含中葡对照、优先级、匹配度 |
| 免责声明 | 简报末尾全文，导出底稿时同样包含 |

### 请注意

- **「暂不可自动定稿」** 表示多条未过门控，仍可进入复核，但对外使用前必须由法务重点审阅 blocked 条目。  
- 智能润色失败时，系统会**自动回退**为模板模式，不影响继续复核。  
- 葡萄牙语部分供当地律师或合作方阅读，**不能替代**正式葡语法律意见书。  
- 简报页顶部的黄色提示栏会再次提醒门控规则，请务必阅读。

## 步骤 5 · 业务提交 / 法务复核（关键环节）

### 业务：提交法务复核

**入口：** 简报页 → **「提交法务复核」**

| 您需要做什么 | 说明 |
|--------------|------|
| 确认简报已生成 | 可先使用「仅模板」即时生成，或等待 LLM 润色完成 |
| 点击提交 | 场景状态变为「待法务复核」 |

**请注意：** 业务账号**无法**定稿或导出底稿，这是合规设计。

### 法务：复核与定稿

**入口：** 工作台「待处理任务」→ **开始复核**，或简报页 → **「进入法务复核」**

法务待办按 **复核优先级** 排序（门控未过线、需外聘标记的条目会排在前面）。可使用顶部筛选：**全部 / 待处理 / 门控未过线 / 需外聘**。

---

### 中国法务能审什么、不能审什么

本环节是 **协查级专业把关**，不是对中国法务做巴西实体法考试。您审的是：**系统把「这条核查事实 + 中文简报 + 法条引用」绑在一起之后，这套表述是否站得住、是否该交给外聘**。

| 中国法务**可以**判断 | 中国法务**不应替代** |
|----------------------|----------------------|
| 中文简报与中文法条摘要是否说的是**同一件事**（明显张冠李戴、答非所问） | 巴西法条在特定事实下的**正式法律结论** |
| 法条标题/摘要是否**过泛**（如总则、原则性条款），无法回答本条具体问题 | 当地律师应出具的**葡语法律意见书** |
| 简报表述是否**过度确定**（把「可能需关注」写成「必然违法」） | 仅凭匹配度分数推断「还缺哪几条法」 |
| 匹配度 **≤75 分** 或信息明显不足时，是否应 **勾选「需外聘」** 或 **驳回** | 未读侧栏法源片段就一键「全部确认」 |

**重要说明：** 匹配度（如 73 分）是系统检索相关性的**谨慎信号**，不能据此反推「还缺哪几条法」。若对贴题性有疑，应 **驳回** 并在批注中说明，或勾选 **需外聘巴西律所**。

---

### 协查三问（建议逐条对照侧栏）

点击某条核查项右侧 **「查看简报片段」**，侧栏会展开 **简报摘录 + 法源对照 + 门控说明**，无需离开复核页。逐条自问：

| 序号 | 问题 | 若答案为「否」或「不确定」 |
|------|------|---------------------------|
| **1** | 中文简报和中文法条摘要说的是**同一件事**吗？ | 倾向 **驳回**；批注写明「简报与引用不对题」 |
| **2** | 法条标题是否**过泛**，无法回答这条具体问题？ | 倾向 **驳回** 或勾选 **需外聘** |
| **3** | 仍有疑问，或匹配度偏低（如 **≤75**），是否应交给巴西当地律师？ | 勾选 **需外聘**，必要时 **驳回** |

三问均通过、且您对表述无异议时，点击 **确认**。

若已有条目 **驳回** 且需业务补充材料，可点击 **「退回业务补充」**（须至少 1 条已驳回）。业务将在**同一项目**修改后重新提交，无需新建场景。

---

### 您需要做什么

对 **每一条** 核查项完成以下操作之一：

| 操作 | 适用情况 |
|------|----------|
| **确认** | 三问通过；认同核查要点与法条引用**方向**（不要求您论证巴西法实体结论） |
| **驳回** | 不认同本条**系统协查结论**（简报与引用不对题、信息不足、过度确定等） |
| **批注** | 补充说明；**驳回时建议必填**（如「需外聘补充环评意见」） |
| **需外聘** | 勾选后标记该条须巴西当地律所出具正式意见；可与确认或驳回组合使用 |

**驳回的对象：** 是 **单条** 系统协查结论（事实绑定 + 简报 + 引用），**不是** 驳回业务人员，也 **不是** 整份项目作废。

全部条目处理完后：

1. 可使用 **「全部确认」** 快速通过（**仅当您已逐条看过侧栏且无异议时**）。  
2. 点击 **「提交复核定稿」** 锁定本次协查结论。  
3. 定稿后点击 **「导出 Word · 法律研究意见书」** 或 **「导出 PDF」** 下载文件。

### 完成后您会看到什么

- 每条状态变为「已确认」「已驳回」或「待复核」。  
- 顶部统计：已确认 / 已驳回 / 待处理数量。  
- 定稿后状态锁定，**不可再修改**单项结论（需重新发起协查任务）。  
- 导出文件命名：Word 为 `{项目名称}_法律研究意见书.docx`；PDF 为 `{项目名称}_协查底稿.pdf`。  
- Word 含：抬头、文号、致/自/关于表、分维度意见、综合结论、复核记录、免责声明与签章栏；PDF 为 legacy 协查底稿结构。

### 请注意

- **这是整个流程中唯一应由持证法务人员签字负责的环节。**  
- 存在「待复核」条目时，**无法定稿**，也无法导出。  
- 「全部确认」不会替您阅读内容，请勿在未打开侧栏审阅的情况下一键通过。  
- 被驳回或门控未过线的条目，应在批注中说明后续动作（如「需外聘巴西律所补充环评意见」）。  
- 定稿前可点击 **「查看简报」** 返回核对；定稿后仅可导出，不可回改。

**驳回之后（当前版本）：**

| 现象 | 说明 |
|------|------|
| 条目状态 | 变为「已驳回」，您的批注会保留在复核记录中 |
| 定稿与导出 | 须 **处理完所有条目**（确认或驳回）才能定稿；底稿可含已确认 / 已驳回 / 部分通过条目 |
| 业务端 | 进度页 **「法务反馈」** 可查看驳回批注与「需外聘」标记 |
| 系统行为 | 法务可 **「退回业务补充」**；业务在**同一项目**修改后 **「保存并重新提交法务」**，无需新建场景 |

若贵司制度要求「驳回须退回业务修改同一项目」，请在批注中写清原因，并通知业务同事补充材料。

## 步骤 6 · 导出与归档

### 您需要做什么

在复核页定稿后，选择 **导出 Word · 法律研究意见书**（便于二次编辑）或 **导出 PDF**（便于传阅定版）。

### 完成后您会看到什么

- 浏览器下载 Word 意见书或 PDF 底稿文件。  
- 工作台该场景状态更新，可随时从「最近协查场景」再次进入查看（已定稿不可改结论）。

### 请注意

- 导出底稿仍附带**免责声明**，对外报送前请按贵司制度加盖「仅供内部参考」等标识。  
- Word 版适合插入律师正式意见、补充附件；PDF 版适合存档与邮件分发。  
- 底稿中的法条链接在 Word 中可点击；PDF 中链接是否可点取决于阅读器。  
- 如需多版本对比（模板版 vs 润色版），请在定稿前分别生成简报并自行命名存档。

## 步骤 7 · 法规动态监测（可选）

**入口：** 工作台 → **「法规动态监测（辅线）」**

### 您需要做什么

点击 **「手动扫描法规更新」**，查看是否有与本地法源库相关的变更提醒。

### 完成后您会看到什么

- 若无更新，提示「暂无监测提醒」。  

### 请注意

- 当前为 **MVP 能力**：主要检测平台内置法源语料的变更，**不能替代**对 LexML 等官方法源的持续跟踪。  
- 正式投产场景应配合专人或律所订阅法规更新服务。  
- 监测结果不会自动修改您已定稿的导出文件；若发现重大变更，应**重新发起协查**。

## 步骤 8 · 法源语料维护（法务专属）

当需要**在平台内补充或更新法条片段**（供后续协查检索绑定），请使用本功能。这与「复核某次协查结论」不同：这里改的是**全平台共享的法源库**。

**入口（任选其一）：**

- 顶部导航 → **「法源维护」**  
- 工作台 → 法规动态监测区域 → **「法源语料维护」**

> 仅 **法务 / 管理员** 账号可见；业务账号无此入口。

---

### 8.1 先分清：法源语料 vs 核查规则

| 维护对象 | 在平台里做什么 | 谁通常负责 |
|----------|----------------|------------|
| **法源语料** | 在本页新增/编辑/删除法条片段，绑定核查项编号 | 法务录入 + 实施复核 |
| **核查规则** | 新增 IBAMA 检查点等**清单条目** | 实施人员更新规则库（本页暂不支持） |
| **单次协查** | 复核页批注、驳回、导出 | 法务按项目操作 |

协查检索时，系统用 **法源语料** 为每条核查项匹配 Top-3 法条；`checklist_codes`（如 `LAB-001`）必须与核查清单编号一致，否则无法精准绑定。

---

### 8.2 补充或更新法条的标准流程

```
① 在 LexML / STF / STJ 等官方法源确认最新文本、URN 与链接
        ↓
② 登录法务账号 → 法源维护 → 「新增法条条目」或编辑已有条目
        ↓
③ 填写葡语/中文标题、原文片段、维度、关联核查项编号等
        ↓
④ 保存后点击 「重建法源索引」（必做，否则新协查仍用旧索引）
        ↓
⑤ 可选：工作台「手动扫描法规更新」确认监测提醒
        ↓
⑥ 抽样发起新协查，检查法条绑定与匹配度是否正常
        ↓
⑦ 若法规变化影响已定稿项目 → 通知业务重新协查（底稿不会自动变更）
```

---

### 8.3 页面操作说明

| 操作 | 说明 |
|------|------|
| **筛选** | 按维度、法源类型（lexml/stf/stj/jusbrasil）或关键词搜索 |
| **新增法条条目** | 打开表单；条目 ID 可留空由系统生成 |
| **编辑 / 删除** | 修改原文、标题、关联核查项等；删除前请确认无项目依赖该引用 |
| **重建法源索引** | 保存任何语料变更后**必须执行**，检索与简报才会使用新内容 |
| **语料版本** | 每次增删改会自动升版本号（如 1.2 → 1.3），便于追踪变更 |

**表单必填要点：**

- 葡语标题、中文标题、URN、官方链接  
- 合规维度（劳工 / 外资 / 税制 / 行业准入）  
- 葡语原文片段 + 中文摘要  
- **关联核查项**（逗号分隔，如 `LAB-001, SEC-012`）— 可从下拉提示中选择规则库已有编号  

---

### 8.4 什么情况下用本功能、什么情况下不用

| 场景 | 建议做法 |
|------|----------|
| 官方法规更新，需纳入后续协查检索 | **法源维护** 新增/修订 + 重建索引 |
| 仅本次项目补充说明 | **复核页批注**，或导出 Word 后由律师插入正式意见 |
| 需要新增核查清单条目（新检查点） | 联系实施人员更新规则库，再在本页绑定法条 |
| 已定稿底稿需反映新法 | **重新发起协查**；不会自动改写历史底稿 |

---

### 8.5 请注意

- 录入内容须来自**可溯源官方法源**；平台不自动从 LexML 拉取全文。  
- 重建索引可能需要数秒；期间可继续复核，但**新提交协查**建议在索引完成后进行。  
- 变更会写入系统**审计日志**（操作人、条目 ID、语料版本）。  
- 本功能提供协查级法条片段管理，**不构成**对巴西法的正式法律意见；对外仍须当地律师定稿。

## 五、全流程一览

```
登录 & 确认免责
    ↓
提交协查场景 ──→ 按子赛道生成专项核查清单（BYD 约 24 条）
    ↓
法源自动检索 ──→ 每条绑定 Top-3 法条 + 匹配度
    ↓

    ↓
法务逐条复核 ──→ 确认 / 驳回 / 批注
    ↓
提交定稿 ──→ 导出 Word 法律研究意见书 / PDF 协查底稿
```

| 阶段 | 您得到的价值 | 必须人工把关的点 |
|------|----------------|------------------|
| 场景提交 | 结构化协查范围 | 描述是否完整、维度是否选对 |
| 清单 | 不漏项的检查列表 | 条目是否适用于您的具体交易结构 |
| 法条检索 | 可溯源的初步依据 | 匹配度 < 70、未命中条目 |
| 简报 | 对内对外沟通底稿 | blocked 条目、葡语表述 |
| 复核定稿 | 可归档的正式过程记录 | 每条确认/驳回的理由 |
| 导出 | Word 意见书 + PDF 底稿 | 是否需律师加盖正式意见 |

## 七、常见问题（客户版）

**Q：系统给出的风险等级可以直接写进董事会材料吗？**  
A：不可以直接作为最终结论。仅可作为协查线索，须经法务及外部律师复核后使用。

**Q：为什么有些条目显示「需法务复核」？**  
A：法条匹配度低于 70 分，或未检索到足够依据。系统故意阻止其进入「自动定稿」状态。

**Q：智能润色和「仅模板」差别大吗？**  
A：模板版结构完整、即时可用；润色版中葡表述更自然，但不改变法律结论与法条引用。未配置 API 时仅模板可用。

**Q：可以跳过复核直接导出吗？**  
A：不可以。这是产品的合规设计，确保每份底稿都有明确的人工确认记录。

**Q：一个账号可以做多个项目吗？**  
A：可以。每个项目在工作台「最近协查场景」中独立保存。

**Q：定稿后发现错误怎么办？**  
A：定稿后当前任务锁定。请新建一次协查场景重新走流程，或在 Word 底稿中按贵司制度做修订版管理（须注明修订原因）。

**Q：中国法务看不懂葡萄牙语，怎么审法条？**  
A：请对照侧栏中的 **中文法条摘要** 与 **中文简报**，用「协查三问」判断贴题性；需核对原文时可点击溯源链接。实体法结论仍须外聘巴西律师。

**Q：匹配度 73 分说明缺了哪几条法？**  
A：**不能** 据此推断。分数只表示检索相关性偏低，应视为谨慎信号：打开侧栏核对，不确定则驳回或勾选需外聘。

**Q：驳回是驳回业务同事吗？**  
A：不是。驳回的是 **本条系统协查结论**（简报 + 引用绑定是否可用）。批注中说明专业理由即可，并线下与业务沟通后续动作。

**Q：驳回后业务能看到我的批注吗？**  
A：可以。业务在 **进度页 → 法务反馈** 可看到驳回条目与批注、「需外聘」标记；工作台列表也会显示「待跟进」提示。导出底稿中同样含复核记录。

**Q：「需外聘」和「驳回」有什么区别？**  
A：**需外聘** 标记该条须当地律师正式意见，可与确认同时使用（协查线索可用，但对外结论须外聘）。**驳回** 表示您不认可本条系统结论，批注应说明原因。

**Q：数据存在哪里？会不会用于训练 AI？**  
A：默认仅用于本次协查处理，不用于对外模型训练。具体部署方式（本地 / 私有云）请咨询管理员。
