# Vela 出海法务平台 · 操作手册

**版本：** MVP 1.0  
**适用场景：** 巴西涉外投资合规协查（拉美首发 · 演示：BYD 坎皮纳斯直接投资）  
**文档读者：** 企业法务、合规人员、受邀测试者、演示讲解人员

---

## 1. 产品简介

Vela 出海法务平台是一款面向企业法务部的 Web 应用，用于将**中文投资场景**转化为可复核的**专项核查清单**、**多源法条检索结果**与**中葡双语法律风险简报**，并最终导出**协查底稿**（Word / PDF）。

### 1.1 主流程（8 步）

| 步骤 | 名称 | 说明 |
|------|------|------|
| 1 | 场景输入 | 中文描述投资项目（国家、行业、动作类型等） |
| 2 | 范围确认 | 自动映射 5 大审查维度：劳工 / 外资 / 税制 / 环保 / 行业准入 |
| 3 | 核查项定位 | 规则引擎 + 可选 LLM：在法务选定维度内触发核查条目与子赛道 |
| 4 | 多源 RAG | 从 LexML / STF / STJ / Jusbrasil 精选语料检索 Top-3 法条 |
| 5 | 跨语种推理 | 中英葡关键词联合打分，低于 70 分标记「需法务复核」 |
| 6 | 生成简报 | 中葡双语执行摘要 + 分维度风险说明 |
| 7 | 法务复核 | 逐条确认 / 驳回 / 批注，提交定稿 |
| 8 | 导出归档 | 导出 Word 法律研究意见书 / PDF 协查底稿 |

### 1.2 辅线功能

- **法规动态监测（MVP）：** 工作台手动扫描本地语料变更，查看提醒列表  
- **API 文档：** 后端 OpenAPI，供集成与自动化测试

### 1.3 重要合规说明

- 平台输出**不构成正式法律意见**，须由持证律师复核  
- 匹配度 **低于 70 分** 或未命中法条的条目，**不得自动定稿**  
- LLM 仅用于**润色措辞**，不新增法律结论、法条名称或溯源链接  
- 注册与导出均含**免责声明**，使用前须确认

## 3. 安装与启动

### 3.1 获取代码

私有仓库需被添加为 GitHub Collaborator 后方可克隆：

```bash
git clone https://github.com/so-ki/vela.git
cd vela-platform   # 若克隆目录名不同，请进入实际项目根目录
```

### 3.2 一键启动（推荐）

在项目根目录执行：

```bash
chmod +x scripts/start.sh
./scripts/start.sh
```

脚本将自动完成：

- 释放 8000 / 5173 端口上的旧进程  
- 创建 Python 虚拟环境并安装依赖（含 RAG / Chroma）  
- 复制 `backend/.env.example` → `backend/.env`（若不存在）  
- 初始化数据库与演示账户  
- 同时启动后端与前端

启动成功后，终端会显示：

| 服务 | 地址 |
|------|------|
| 前端页面 | http://127.0.0.1:5173 |
| 后端 API | http://127.0.0.1:8000 |
| API 文档 | http://127.0.0.1:8000/docs |

### 3.3 停止服务

在运行 `./scripts/start.sh` 的终端按 **Ctrl + C**，即可同时停止前后端。

### 3.4 Docker 启动

**开发栈（SQLite）：**

```bash
docker compose up --build
```

**生产栈试部署：**

```bash
cp .env.production.example .env.prod
# 编辑 SECRET_KEY、POSTGRES_PASSWORD 等
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d --build
```

或一键冒烟（自动生成本地 `.env.prod.smoke`，端口默认 8089）：

```bash
chmod +x scripts/prod_smoke.sh
./scripts/prod_smoke.sh
```

详见 [DEPLOYMENT.md](./DEPLOYMENT.md)。

## 5. 快速演示（约 5 分钟）

适合比赛演示、领导汇报或首次体验。

1. **登录** → 进入工作台  
2. 点击 **「一键生成完整样本」**  
   - 系统自动完成：BYD 场景 → 约 24 条清单（5 子赛道）→ 法源绑定 → 双语简报 → 进入复核页  
   - 若已配置 LLM Key，简报会尝试通义千问润色（约 1–2 分钟）  
3. 在**法务复核**页点击 **「全部确认」**  
4. 点击 **「提交复核定稿」**  
5. 点击 **「导出 Word · 法律研究意见书」** 或 **「导出 PDF」** 下载定稿文件  

默认 Word 为**法学院格式《涉外投资合规法律研究意见书》**，文件名 `{项目名称}_法律研究意见书.docx`；PDF 仍为 legacy 协查底稿格式。定制方法见 **第 9 章**。

仓库根目录还预置了两份 Word 样本（无需启动服务即可查看排版效果）：

- `BYD坎皮纳斯_协查底稿_样本.docx` — 规则模板版  
- `BYD坎皮纳斯_协查底稿_LLM润色.docx` — LLM 润色版

## 6. 分步操作指南

若需展示完整业务链路，可按以下顺序手动操作。

### 6.1 提交协查场景（Step 1–2）

**入口：** 工作台 → **「提交协查场景」** 或 **「使用演示模板」**

| 操作 | 说明 |
|------|------|
| 加载演示模板 | 自动填入 BYD 坎皮纳斯项目字段（项目名称、地点、行业、投资结构等） |
| 选择审查维度 | 默认勾选 5 维度，至少保留 1 项 |
| 填写中文描述 | 说明投资背景、产能、用工规模等，便于核查项定位与 RAG 检索 |
| 提交 | 系统按**子赛道**生成专项核查清单（BYD 演示约 **24 条**），跳转至清单页 |

**演示模板要点：**

- 国家：巴西 · 圣保罗州 · 坎皮纳斯  
- 行业：新能源  
- 动作类型：直接投资与新建项目（Greenfield / FDI）

### 6.2 查看核查清单与法源绑定（Step 3–4）

**入口：** 清单页（提交场景后自动进入，或从工作台「最近协查场景」→ **清单**）

页面展示：

- 5 大维度分组（劳工 / 外资 / 税制 / 环保 / 行业准入）；全场景清单条目随子赛道与勾选维度动态生成  
- 每条含：编号、优先级、纳入理由、清单相关度  
- **相关法条（Top 3）：** 来源标签（LexML / STF / STJ / Jusbrasil）、匹配度、摘录、溯源链接  

**操作要点：**

1. 等待页面自动完成法源检索（首次可能需数秒）  
2. 点击 **「溯源：LexML / 法院门户 ↗」** 可在新标签页打开官方法源  
3. 匹配度 **低于 70 分** 的条目会标注 **「需法务复核」**  
4. 检索完成后，点击 **「生成双语简报」** 进入下一步  

### 6.3 生成双语法律风险简报（Step 5–6）

**入口：** 简报页

页面展示：

- 中葡双语标题与执行摘要  
- 分维度综合风险等级（较高 / 中等 / 可控）  
- 每条核查项：门控状态、匹配度、中葡风险说明、法条依据与溯源  
- 未通过门控的条目汇总在 **「需法务复核清单」** 区域  
- 页底**免责声明**全文  

**操作按钮：**

| 按钮 | 说明 |
|------|------|
| LLM 润色生成 | 调用通义千问润色中葡表述（需 API Key，约 60–120 秒） |
| 仅模板 | 不调用 LLM，使用规则模板汇编（即时完成） |
| 进入法务复核 | 跳转复核工作台 |

**状态说明：**

| 状态 | 含义 |
|------|------|
| 可提交法务复核 | 门控通过，可进入复核 |
| 部分条目需复核 | 存在 blocked 条目，仍可复核但须人工关注 |
| 暂不可自动定稿 | 多条未过门控，建议补充检索或外聘律所 |

未配置 API Key 时，系统自动使用**规则模板模式**，不影响主流程演示。

### 6.4 法务复核与定稿（Step 7）

**入口：** 复核页

**逐条操作：**

| 操作 | 说明 |
|------|------|
| 确认 | 认可该条核查结论 |
| 驳回 | 不认可，须补充说明 |
| 批注 | 在输入框填写意见，失焦后自动保存 |

**批量操作：**

1. **全部确认** — 一键将所有待处理项设为已确认  
2. **提交复核定稿** — 锁定复核结果（定稿后不可再修改单项）  
3. **导出 Word / 导出 PDF** — 定稿后可用；Word 默认 `{项目名称}_法律研究意见书.docx`，PDF 为 `{项目名称}_协查底稿.pdf`  

**定稿条件：** 所有条目均须已「确认」或「驳回」，且无待处理项。

### 6.5 返回工作台

定稿并导出后，可点击 **「返回工作台」**。工作台 **「最近协查场景」** 列表中可再次打开该项目的清单、简报或复核页。

## 7. 工作台其他功能

### 7.1 系统状态（法务 / 管理员）

工作台 **法务账号** 可见，用于确认后台服务就绪：

- 数据库连接状态  
- 规则库（巴西 · 投资协查 · 5 维度）  
- 法源索引条数与检索模式（keyword / vector）

业务账号不展示此面板；提交失败时联系管理员排查。

### 7.2 法规动态监测（辅线）

1. 查看当前监测提醒（若有语料变更）  
2. 点击 **「手动扫描法规更新」** 触发扫描  
3. 扫描结果会更新提醒列表与法源索引状态  

> MVP 阶段为本地语料变更检测；生产环境可对接 LexML 增量同步与订阅推送。

### 7.3 法源语料维护（法务 Web 管理）

**入口：** 顶部导航 **「法源维护」** 或工作台 → **「法源语料维护」**（仅 `legal` / `admin`）

| 操作 | 说明 |
|------|------|
| 列表筛选 | 按维度、法源类型、关键词搜索 |
| 新增 / 编辑 / 删除 | 维护 `brazil_legal_corpus.json` 等价数据（Web 写入） |
| 重建法源索引 | 保存后必点，调用 `POST /legal/corpus/reindex` |
| 审计 | 增删改与重建索引写入 `audit_log` |

核查规则库（`brazil_new_energy.json`）仍须实施人员维护；语料表单中的 `checklist_codes` 须与规则库编号一致。

### 7.4 快捷链接

- **API 文档 /docs** — 打开 Swagger 交互文档  
- **GitHub 仓库** — 项目源码

## 8. LLM 润色配置（可选）

### 8.1 配置步骤

1. 编辑 `backend/.env`（可从 `backend/.env.example` 复制）  
2. 填入通义千问 Key：

```env
QWEN_API_KEY=sk-你的密钥
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_MODEL=qwen-plus
LLM_POLISH_ENABLED=true
```

3. 重启后端（重新运行 `./scripts/start.sh`）  
4. 访问 http://127.0.0.1:8000/api/v1/llm/status 或在简报页查看 LLM 状态  

### 8.2 获取 API Key

登录 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/) → 创建 API Key → 粘贴至 `.env`。

### 8.3 注意事项

- **勿将 API Key 提交至 Git** 或粘贴至公开 Issue  
- 润色超时已设为 180 秒；若失败，系统自动回退模板模式  
- DeepSeek 亦支持，详见 `backend/.env.example` 注释

## 9. Word 导出模板定制

本章面向**实施人员、运维与定制开发**，说明如何调整定稿后导出的 Word 文档（默认：法学院《涉外投资合规法律研究意见书》）。PDF 导出仍使用 legacy 协查底稿格式，若需同步定制须另行修改代码（见 9.5）。

### 9.1 模板机制概览

| 项目 | 说明 |
|------|------|
| 默认模板 | `law_school` — 法学院标准法律研究意见书 |
| 旧版模板 | `legacy` — 早期「拉美投资合规协查底稿」 |
| 切换方式 | 环境变量 `EXPORT_TEMPLATE` |
| 数据来源 | 场景字段 + 核查清单 + 双语简报 + 法务复核结果 |
| 前端按钮 | 复核页显示「导出 Word · 法律研究意见书」（标签来自 `/export/config`） |

**核心代码路径（项目根目录 `vela-platform/`）：**

| 文件 | 作用 |
|------|------|
| `backend/app/services/export_template_law_school.py` | 法学院 Word 正文结构与排版 |
| `backend/app/services/export_service.py` | 按 `EXPORT_TEMPLATE` 路由到对应模板 |
| `backend/app/services/export_context.py` | 从场景聚合导出所需数据 |
| `backend/app/services/disclaimer.py` | 附录免责声明全文 |
| `backend/app/core/config.py` | 导出相关环境变量定义 |
| `backend/app/api/system.py` | `GET /export/config` 返回当前模板标签 |

### 9.2 第一档：仅改抬头与机构信息（不改代码）

编辑 `backend/.env`（生产环境为 `.env.prod` 或 Docker 环境变量），修改后**重启后端**：

```env
EXPORT_TEMPLATE=law_school
EXPORT_ORG_NAME=某某大学法学院 · 企业合规研究中心
EXPORT_ORG_DEPARTMENT=涉外投资合规研究项目组
EXPORT_RECIPIENT_LABEL=企业管理层 / 投资委员会
```

上述变量会写入 Word 的：

- 页眉**机构名称**、**部门名称**
- 信息表中 **「致」**、**「自」** 等字段

生产 Docker 部署时，在 `.env.prod` 中配置同名变量，执行 `docker compose ... up -d --build` 重建 backend 服务。详见 [DEPLOYMENT.md](./DEPLOYMENT.md)。

### 9.3 第二档：改章节大纲、版式与文案（改 Python 模板）

打开 `backend/app/services/export_template_law_school.py`，按下列对照表定位修改点：

| 想改什么 | 修改位置（函数 / 区域） |
|----------|-------------------------|
| 文档主标题 | `build_law_school_docx()` 内 `"涉外投资合规法律研究意见书"` |
| 文号规则 | `_doc_number()`（默认 `VELA-YYYY-#####`） |
| 页边距、默认字体 | `build_law_school_docx()` 开头 `section.*_margin`、`_set_run_font()` |
| **章节大纲**（增删改「一、二、三…」） | 各 `_add_heading(doc, "一、…")` 段落及对应正文块 |
| 致 / 自 / 关于 信息表字段 | `_cover_table(...)` 的 `rows` 列表 |
| 每条核查项展示内容 | 「二、分维度法律核查意见」循环（法源、风险说明、批注） |
| 综合结论、复核意见 | 「三、…」「四、…」段落 |
| 签章栏 | 文末 `sig_table` |
| 下载文件名后缀 | `safe_export_filename(..., "法律研究意见书.docx")` |
| 重要说明 / 固定免责声明段落 | `build_law_school_docx()` 内硬编码说明文字；附录全文来自 `disclaimer.py` |

**当前默认大纲（可按律所样张调整顺序或标题）：**

1. 抬头（机构名 + 部门）  
2. 主标题 + 文号 + 出具日期  
3. 致 / 自 / 关于 等信息表  
4. 【重要说明】  
5. 一、项目事实概要  
6. 二、分维度法律核查意见（按维度 → 核查项 → 法源 → 风险说明）  
7. 三、综合结论与风险提示  
8. 四、法务复核意见  
9. 附录 · 免责声明  
10. 经办 / 复核 / 日期签章栏  

**导出时可用的主要数据字段：**

| 变量 | 内容示例 |
|------|----------|
| `scenario_obj` | 项目名、城市/州/国家、描述、投资结构、用工规模 |
| `payload` | 协查法域名、子赛道、清单 `sections` |
| `brief` | 中葡摘要、分条风险说明、`blocked_items` |
| `review` | 复核人、定稿时间、每条 `decision` / `comment` / 外聘标记 |
| `sections_legal` | 带法源命中与匹配度的清单条目 |

修改后验证步骤：

1. 重启 `./scripts/start.sh`  
2. 登录法务账号，走完 **一键样本 → 定稿**  
3. 复核页点击 **导出 Word**，检查排版与字段  
4. 可选：运行 `./scripts/verify_e2e.sh` 确认主流程未破坏  

### 9.4 第三档：新增律所 / 院校专用模板（推荐）

不宜长期直接改 `law_school.py`，建议**复制为新文件**：

1. 复制 `export_template_law_school.py` → 如 `export_template_firm_abc.py`  
2. 实现 `build_firm_abc_docx(scenario) -> tuple[bytes, str]`  
3. 在 `export_service.py` 的 `build_docx()` 中增加分支，例如：

   ```python
   if settings.export_template == "firm_abc":
       return build_firm_abc_docx(scenario)
   ```

4. 若需律所专属配置（地址、电话、Logo 路径等），在 `config.py` 增加对应环境变量  
5. 在 `backend/app/api/system.py` 的 `/export/config` 中补充 `docx_label`，前端按钮文案会自动更新  
6. 部署时设置：

   ```env
   EXPORT_TEMPLATE=firm_abc
   ```

切换回旧版协查底稿 Word：

```env
EXPORT_TEMPLATE=legacy
```

### 9.5 其他说明

| 话题 | 说明 |
|------|------|
| PDF | 仍为 `export_service.py` 内 `build_sample_pdf()`（legacy），与 Word 模板独立 |
| 免责声明 | 法学院模板「附录」读取 `DISCLAIMER_FULL_TEXT`；改平台免责声明会同步进 Word |
| 固定 Word 样张 | 若须与律所 `.docx` 样张像素级一致，可后续改为「加载空白 docx 模板 + 填占位符」；当前 MVP 用 `python-docx` 代码拼装 |
| API 查询 | `GET /api/v1/export/config` 返回当前 `template` 与 `docx_label` |

## 10. 常见问题

### 10.1 端口被占用

```bash
lsof -ti:8000 | xargs kill -9
lsof -ti:5173 | xargs kill -9
./scripts/start.sh
```

### 10.2 一键样本生成失败

- 确认后端已启动：访问 http://127.0.0.1:8000/api/v1/health 应返回正常  
- 重新运行 `./scripts/start.sh`  

### 10.3 法源检索失败或 404

- 确认已安装 RAG 依赖（`start.sh` 会自动安装 `requirements-rag.txt`）  
- 重启服务后重试  
- Chroma 向量模型下载失败时，系统回退**关键词检索**，演示仍可继续  

### 10.4 LLM 润色长时间无响应

- 正常耗时约 **60–120 秒**，请耐心等待  
- 未配置 Key 时使用「仅模板」即可  
- 网络不通 DashScope 时会显示回退提示  

### 10.5 无法导出 Word / PDF

- 须先 **提交复核定稿**  
- 确认所有条目已确认或驳回  

### 10.6 登录失败

- 确认使用演示账户：`legal@demo.vela` / `Demo1234!`  
- 若数据库被重置，重新运行 `backend/scripts/seed_demo_user.py`

## 11. 演示讲解建议

### 11.1 15 分钟标准演示脚本

| 时间 | 环节 | 讲解要点 |
|------|------|----------|
| 2 min | 背景 | 中企出海巴西、法务协查痛点、Vela 定位 |
| 1 min | 登录工作台 | 8 步路线图、系统状态、法源 **70 条** |
| 5 min | 一键样本 | 强调 LexML 溯源、70 分门控、中葡双语 |
| 3 min | 法务复核 | 人机协同、确认/驳回/批注、合规闭环 |
| 2 min | 导出底稿 | Word 法律研究意见书 / PDF、免责声明 |
| 2 min | 辅线 + Q&A | 法规监测 MVP、API 开放、Phase 2 规划 |

### 11.2 重点强调的差异点

- 不是「ChatGPT 给法律意见」，而是 **规则 + RAG + 门控 + 人工复核**  
- 每条风险附 **可点击官方溯源**  
- LLM 角色限于 **润色与核查项定位辅助**，结论仍由法务定稿
